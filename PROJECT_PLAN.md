# 🚀 DevOps Master Project: Cloud-Native E-commerce Platform

## 🎯 Цель проекта

Построить **полноценную production-like инфраструктуру** для микросервисного приложения, используя весь стек DevOps-инструментов. Это будет твой **флагманский проект для портфолио**, который покажет работодателям, что ты умеешь работать с реальными системами.

## 🚀 Как будем работать

1. **Я даю задачу** — конкретный шаг с объяснением **ЧТО** делаем
2. **Ты выполняешь** — пишешь код, конфиги, запускаешь
3. **Я объясняю** — **ПОЧЕМУ** именно так и **КАК** это работает в production
4. **Фиксируем** — сохраняем результат в документацию

## ✅ Критерии готовности проекта

Проект считается завершённым, когда:

- [ ] Все сервисы деплоятся одной командой (`make deploy` или `terraform apply`)
- [ ] CI/CD пайелайн зелёный для всех сервисов
- [ ] Мониторинг показывает метрики всех компонентов
- [ ] Алерты приходят в Telegram при проблемах
- [ ] Логи со всех сервисов собираются централизованно
- [ ] Backup работает и restore протестирован
- [ ] Security scanning проходит без критических уязвимостей
- [ ] Документация полная и актуальная

## 🎯 Следующий шаг

Готов начать? Первый этап — **Docker и контейнеризация**.

Нужно:
1. Выбрать стек для каждого микросервиса (язык + фреймворк)
2. Написать простые сервисы (Hello World + health endpoint)
3. Создать Dockerfile для каждого
4. Запустить через docker-compose

**С чего начнём?**
